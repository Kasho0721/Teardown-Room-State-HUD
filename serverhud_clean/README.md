# Room State HUD

[English Version](./README.en.md)


这个目录是一个给 **Teardown 联机房状态采集** 用的 HUD / 脚本模块。

它的作用不是做花里胡哨的 UI，而是把房间关键状态整理出来，方便 bridge 读取并上报到云端。

---

## 作用

它会在游戏运行时维护这些信息：
- 房主名称
- 当前玩家数
- 最大玩家数
- 玩家列表
- 启动时间 / 在线时长
- 快照与变更序号

这些数据会被写入可供 bridge 读取的位置，后续再由 bridge 上传到 `cloud-center`。

---

## 当前命名说明

这个模块原本偏调试用途，所以以前名字比较随便。
现在公开仓库里把展示名整理成了更贴项目用途的名字：

**Room State HUD**

不过内部读取逻辑为了兼容 bridge，仍然保留了原有的数据路径约定，避免改个名字把整套链路搞断。


---

## 与 bridge 的关系

这一层更像“游戏内数据采集器”。

- 它负责从游戏里拿到联机状态
- `bridge/` 负责把这些状态转成结构化数据并上报云端
- `cloud-center/` 负责统一展示
- `onebots/` 负责聊天入口查询

所以它是整套链路最前面的数据来源之一。

---

## 注意

如果你要改这个模块的内部数据键名、路径前缀、导出结构，最好同时更新 bridge 的读取逻辑。

只改显示名问题不大，
但如果把内部标识也一起改了，bridge 可能就读不到数据了。
